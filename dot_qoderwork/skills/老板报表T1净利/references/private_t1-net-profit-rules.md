## 凭证科目中间表

`doris_dws_voucher_subject_mid` 由 voucher、voucher_detail、accounting_dimension、subject 和 cost_gather_strategy 关联生成。

核心过滤与关联：

- `voucher` 与 `voucher_detail` 通过凭证 ID 关联。
- `voucher_detail` 与 `accounting_dimension` 通过凭证明细 ID 关联。
- 限制凭证状态为已审核。
- 限制账套为业务账套 `4`。
- 通过 `dt + subject_code` 关联 `cost_gather_strategy`。
- 限制 `是否采集/是否获取数据 = 是`。
- 通过科目编码和最新 `dt` 关联 `fms_support.subject`，获取科目名称、科目全名、科目层级、一级/二级科目和余额方向。

主要维度：

- `dt`、业务发生日期、科目编码、科目名称、科目全名、科目层级。
- 一级科目编码/名称、二级科目编码/名称。
- 是否为计提科目。
- 科目拆分类型：店铺、货主、部门、仓库、供应商、分销商或无类型。
- 拆分优先级：店铺 > 货主 > 部门 > 仓库 > 供应商 > 分销商 > 无。
- 核算维度包括供应商、部门、公司、店铺、货主、费用项目、物料、银行账号、往来单位、税率、仓库、仓库功能、员工、分销商和辅助资料维度。
- 余额方向枚举：不控制、借方、贷方。

金额字段：

- 借方金额：新版本按借方本位金额汇总。
- 贷方金额：新版本按贷方本位金额汇总。

## 科目费用分摊

目标表：

- 分摊成功表：`dp_dws.doris_dws_finance_cost_sbjct`
- 分摊失败表：`dp_dws.doris_dws_finance_cost_sbjct_failure`

刷新行为：

- 非摊销数据按凭证科目/detail 最近 45 天更新或删除逻辑处理，并回刷受影响的业务发生日期。
- 摊销数据单独处理上月发生变动的日期，历史规则通常在每月 8 号附近处理。

核心输出维度：

- 业务发生日期。
- 科目编码、科目全名、科目层级、一级科目、二级科目。
- 费用编码、费用名称、费用归属。
- 事业群、部门、货主、货主名称。
- 店铺、店铺名称、店铺类型。
- 仓库、仓库名称、仓库功能。
- 国别。
- 一级到四级类目。
- 店铺所属平台、销售模式、供应商编码。
- 货品类别、SPU/货品编码、SKU/规格编码。
- 平台链接、平台规格、平台商品编码。
- 分销商。
- 是否自研品、是否淘汰品、是否停止下单。

2026 优化中的费用来源标识：

- `0`：费用账单 detail。
- `1`：凭证科目中间表。
- `2`：单据费用 index。
- `3`：薪酬费用。

正负号规则：

- 如果费用采集策略中的费用归属是收入，输出金额为正数。
- 如果费用归属是成本或费用，输出金额为负数。

`cost_share_method`：

- `0`：按销售收入分摊。
- `1`：按销售成本分摊。
- `2`：按发货方量分摊。

## 国内费用账单分摊

当金额来源为国内费用明细时：

- 收入取 `income_cost`。
- 支出取 `expend_cost`。
- 限制 `occurrence_type=1`。
- 金额 = 收入 - 支出。
- 排除店铺 `佳帮手分销旗舰店XX【业务公共】`。
- 使用费用发生日作为 `trans_dt`。

国内分摊顺序：

1. 天猫超市特殊逻辑：TMCS 平台来源费用直接按成本分摊，按费用发生日期 + 店铺下的系统单/分销单 SKU 订单成本占比分摊费用。若成本分摊失败，则使用店铺 + 费用发生日期的兜底分摊；仍失败则进入分摊失败表。
2. 其他费用：
   - `cost_type='1'`：订单级费用。通过原始订单号关联系统订单，关联最近 2 个月数据；按原始主单下系统子单 SKU 收入占比分摊。
   - `cost_type='0'`：店铺级费用。通过费用发生日期 + 店铺关联；按该店铺发货当日各 SKU 实际收入占比分摊。
   - `cost_type='2'`：链接级费用。费用先分摊到 SPU/平台链接，再通过费用发生日期 + 平台链接关联订单，按 SKU 收入占比分摊。
   - `cost_type='3'`：订单 + 链接级费用。通过原始订单号 + SPU/平台链接关联最近 2 个月系统单，再按 SKU 收入占比分摊。
3. 链接级兜底：按费用发生日期、平台货品 ID、平台规格 ID、平台商品编码、供应商关联 `doris_dim_prod_spu_sku`，对 goods_code 去重后均分。
4. 店铺日期兜底：使用店铺 + 费用发生日期关联平台链接维表，按可用 SKU 集合拆分。
5. 如果所有分摊路径都失败，写入 `doris_dws_finance_cost_sbjct_failure`。

平台链接维表过滤条件：

- goods_code 启用。
- 货品状态为正常货品。
- 状态为已上架。
- 货品类别为销售货品或配件。

## 跨境费用分摊

跨境逻辑以 2025-08 跨境相关工作表为准。

关键差异：

- 增加跨境系统订单 index 和 `expense_detail_cb`。
- 跨境费用 detail 金额同样使用 `income_cost - expend_cost`，并限制 `occurrence_type=1`。
- 跨境订单关联窗口为 3 个月。
- 跨境系统订单限制包含订单功能 `1` 一般交易订单和 `10` 快递使用单。
- 销售模式包含 `1` 非销售业务、`2` 线上直销、`6` 线下零售/直销。
- 发货状态必须为已发货。
- 排除产销事业群。
- 计算跨境指标时，去掉 `is_cross_border_platform = no` 限制。
- 国内数据国别为国内；跨境数据国别为国外。

跨境分摊层级与国内逻辑一致：

- 订单级按原始订单号分摊。
- 店铺级按费用发生日期 + 店铺分摊。
- 链接级按费用发生日期 + 平台链接分摊。
- 先兜底到平台链接/SKU/供应商映射，再兜底到店铺日期映射。

## 收入、成本与供应商分摊

使用供应商采购占比表将 SKU 分摊到供应商：

- 如果 SKU 金额为 100，供应商采购占比分别为 40% 和 60%，则输出两行：SKU + 供应商 A = 40，SKU + 供应商 B = 60。
- 如果未关联到采购占比关系，使用发货/订单/index 来源中的供应商。
- 默认分摊至第一个供应商；如果第一个供应商是瑕疵品供应商 `G100197`，则顺延到下一个供应商。

销售收入：

- 当 `sales_model = 线上分销(代销)` 时，使用 `estimate_brand_quotation`。
- 其他销售模式在新版本中使用 `sales_amount`；旧版本中曾提到已付金额。
- 不减退款收入按旧 V2 逻辑叠加退款相关金额字段。

成本：

- 含税净利成本根据版本口径使用 `product_cost` 或 `purchase_cost`。
- 不含税净利成本使用 `no_tax_product_cost`。
- 成本也按供应商采购占比分摊。

发货方量：

- 当费用采集策略要求按发货方量分摊时，使用 SKU 发货体积/方量。
- 方量同样按供应商采购占比分摊。

国内收入/成本/方量分摊基础的常见过滤：

- 排除产销事业群。
- 国内指标排除跨境平台；计算跨境指标时去掉该限制。
- 排除店铺所属平台 `班牛`。
- 排除店铺 `佳帮手分销旗舰店XX【业务公共】`。
- 发货状态必须为已发货。

系统订单来源常见限制：

- 订单功能：`1` 一般交易订单、`2` 差价交易订单、`8` 补邮费/补差价订单。
- 销售模式：`2,3,4,6,7`。
- 一般交易订单要求销售出库单不为空，仓库 `WH0026` 除外。
- 旧逻辑中要求退款状态为无退款。

分销发货单来源常见限制：

- 订单功能为 `1`。
- 销售模式为 `7,8,9`。
- 发货状态为已发货。

## 具体费用项

### 物流与包装

- `销售费用_卖家运费_自发费`：取系统订单预估物流费/子单字段 `sub_logistics_cost_fee`，再按 SKU 采购占比分摊到供应商。
- `销售费用_卖家运费_代发费` / `销售费用_仓储服务费`：逻辑同上，字段取 `sub_agent_delivery_fee`。
- `销售费用_包装耗材费`：取子单粒度预估耗材费，再按供应商采购占比分摊。
- 跨境物流费在工作簿备注中当前配置为 0，规则配置前预期通过凭证导入处理。

### 业务协作费

- `销售费用_业务协作费` / 科目 `6601.45`：SA 成本。
- 来源为发货 V1/订单逻辑。
- 备注中说明以净利不含税成本作为基础。
- 订单功能通常限制为 `5`。

### 退货挽损与售后补贴

- `销售费用_退货损失_退货挽损`：销售收入已减退款金额乘以分销商损失挽回比例。
- `销售费用_售后补贴`：销售收入已减退款金额乘以分销商售后补贴比例。
- 比例来源：`sms_product.distributor`。
- 如果收入为负数，按工作簿说明保留负号处理。

### 管理费用税金

- `管理费用_税金`：用收入/成本/体积基础乘以不同店铺税金比例进行分摊。
- 后续版本备注说明该业务规则变更为应用层计算。
- 如果收入为负数，保留负号处理。

### 职工薪酬

- `管理费用_职工薪酬`：
  - 使用业务上传临时表，以及系统订单、分销发货单、跨境订单来源。
  - 先计算 SKU 收入/成本/体积基础。
  - 较新版本按发货日期 + 店铺分摊；较早版本曾按货主分摊。
  - 本月每日人工费用 = 临时表上月费用 / 本月天数。
  - 如果当前月份数据缺失，使用最新上传月份，并除以目标月份天数。
  - 示例：1 月用 1 月数据；2 月没有数据时使用 1 月费用 / 2 月 28 天；待 2 月数据上传后再回刷。

### 上传费比/费用金额类费用

包含：

- `分摊费用`
- `销售费用_供应链管理费`
- `销售费用_职能管理费_宿舍费用`
- `管理费用_职能管理费_宿舍费`
- `销售费用_职能管理费_工位费`
- `管理费用_职能管理费_工位费`
- `管理费用_职能管理费_直播间使用费`
- `销售费用_分摊费用_影棚使用费`
- `销售费用_分摊费用_电服管理费/物流服务费`
- `销售费用_分摊费用_客服管理费/客服服务费`
- `销售费用_分摊费用_售后服务费`
- `销售费用_平台费用_平台佣金`
- `管理费用_品牌宣传费`
- 跨境版本新增 `销售费用_仓储服务费`。

通用逻辑：

- 使用发货/订单来源和业务上传临时表。
- 先计算 SKU 收入/成本/体积。
- 再按发货日期 + 店铺筛选，使用 SKU 基础乘以上传的店铺粒度费比值。
- 如果上传了费用金额，则使用费用金额。
- 如果费用金额和费比值同时存在，费用金额优先。
- 如果只上传费比值，则按费比计算。
- 研发费用是例外：必须上传费用金额；缺少费用金额时应校验失败。

### 集团管理费用

- 使用系统订单、分销发货单、适用时的跨境订单，以及业务上传临时表。
- 先计算 SKU 收入/成本/体积基础。
- 按发货日期 + 店铺分摊。
- 本月每日店铺集团管理费用 = 临时表上月费用 / 本月天数。
- 如果当前月未上传，使用最新上传月份，并除以目标月份天数。

## 天猫超市优化

2025-08-06 优化：

- 如果财务在 T+1 净利其他费用数据中上传费用金额，则使用费用金额。
- 如果财务同时上传费用金额和费比值，费用金额优先。
- 如果财务只上传费比值，则按费比计算。
- 研发费用必须使用费用金额；缺失费用金额时应校验失败。
- 界面需要展示失败原因，并支持下载导入失败文件。
- 天猫超市或其他店铺没有收入、体积、成本基础时，按系统订单已发货数量占比分摊到 SKU。
- 如果也没有发货数量，但发货 V1 中存在该店铺对应 SKU，则将日均费用均分到店铺对应 SKU。

天猫超市示例：

- 店铺月度上传费用金额 = 1200。
- 6 月日均费用 = 1200 / 30 = 40。
- 6 月 1 日 SKU A 数量占比 = 1/4，SKU B 数量占比 = 3/4。
- SKU A 费用 = 40 * 1/4 = 10；SKU B 费用 = 40 * 3/4 = 30。

## 业务上传临时表

空值规则：

- 临时表必须支持上传空值。
- 空值在净利表中默认处理为 0。

上传维度和值：

| 费用类型 | 必传上传粒度 | 上传值 |
|---|---|---|
| 分摊费用 | 月 + 店铺 | 费比；优化后也可上传费用金额 |
| 人工费用 | 月 + 货主；后续也支持月 + 店铺 | 费用金额 |
| 研发费用 | 月 | 仅费用金额 |
| 集团管理费用 | 月 + 店铺 | 费用金额 |
| 销售费用_供应链管理费 | 月 + 店铺 | 费比 |
| 宿舍费、工位费、直播间使用费、影棚使用费、电服/物流服务费、客服/客服服务费、售后服务费 | 月 + 店铺 | 费比 |
| 销售费用_平台费用_平台佣金 | 月 + 店铺 | 费比 |
| 管理费用_品牌宣传费 | 月 + 店铺 | 费用金额 |

重要上传规则：

- 业务侧每月每个费用类型都要按全量货主或店铺上传。
- 后续月份只能增加维度覆盖，不能减少。
- 如果费用或费比为 0，需要显式上传 0。
- 优先取最新月份；取不到时按对应版本说明取最新的上月或最近月份。
- 优化规则：只取上月费用，避免每月 1 到 8 号未导入上月数据时展示异常。

## 差异排查清单

核对或排查 T+1 净利差异时，优先检查：

1. 日期口径：费用发生日期、发货日期、业务发生日期是否混用。
2. 回刷窗口：最近 45 天、上月 8 号回刷，或特殊 5 号/7 号回刷。
3. 国内/跨境过滤：计算跨境指标时是否去掉跨境限制。
4. 排除条件：产销事业群、`班牛`、`佳帮手分销旗舰店XX【业务公共】`。
5. 供应商分摊：采购占比表命中率，以及是否正确兜底到第一个非瑕疵品供应商。
6. 分摊方式：是否按 `cost_share_method` 使用收入、成本或发货方量。
7. 费用来源标识：费用 detail、凭证科目、单据费用 index、薪酬费用是否区分正确。
8. 上传数据：空值转 0、费用金额优先于费比、研发费用仅允许费用金额。
9. 天猫超市：没有收入/成本/体积基础时，是否按发货数量占比兜底。
10. 失败表：无法分摊的数据是否进入 `doris_dws_finance_cost_sbjct_failure`。
