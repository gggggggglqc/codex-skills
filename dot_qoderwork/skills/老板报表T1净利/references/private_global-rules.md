# 全局页面逻辑（17 条）

> 适用于老板报表 / 大屏所有页面，是任何指标的前置约束。

1. **时间口径**
   - 近 30 日：包含昨天的前 30 天
   - 本月：当月 1 号至昨天，不含当天
   - 财年：最近的 4 月 1 号至昨天
   - 数据源：订单表（系统订单 / 退单 / 分销订单 / 跨境订单）；业务日期一般取**发货日期**；费用日期取**账单日期 / 凭证日期**

2. **业态规则**
   - 默认只展示 TOB / TOC，`sales_model in (1, -1)` 与新增销售模式都归入「其他」
   - 国内：`country_type = 1`
   - TOC：`business_type = 2`

3. **2024-01 开始**，供应商大屏的毛利率和成本全部改为**含税含运**口径展示。

4. **不要对比同周期内历史和实时数据**：
   - 实时表每天跑最近 30 天
   - 历史表每天只跑昨日
   - 跑数周期不同会有轻微差异

5. 当前大屏 / 报表中，**未发货占比、已发货占比** 两字段均**未区分销售模式**计算。

6. **趋势图汇总规则**：
   - 全仓库存金额 / 清仓库存金额 / 件均价 / SKU 日产 / 人均日产 / 1 元人力产出 → 月度趋势图按平均值汇总；趋势图汇总值也取平均（月汇总 = 每月平均再求平均）。**例外**：品类 / 货品 / SKU 趋势图汇总值不走平均。
   - 毛利率 / 未发货退款 / 已发货退款 → 天维度汇总取最近 30 天的收入成本求和；月维度汇总取当月每天的收入成本分别求和后再算。
   - **计算 SKU 日产的分子不区分销售模式**。

7. **抽屉默认排序与主列表保持一致**（部门抽屉 = 部门主列表，等等）。

8. **抽屉默认排序字段**：
   - 实时抽屉：默认按昨日金额降序；平台链接表格按收入字段降序；供应商抽屉按销售成本降序；淘汰品抽屉按库存金额降序
   - 历史抽屉：默认按收入字段降序；供应商抽屉按销售成本降序

9. 所有占比值的分母都跟随筛选项联动；汇总行占比永远 100%。

10. 抽屉中只展示**最近 365 日收入 ≠ 0 或今日金额 ≠ 0** 的数据。

11. 弹窗趋势图中的天维度汇总值由**前端**处理。

12. **昨日指标弹窗的最近 30 天趋势图展示真实每日数据，不展示昨日**：例如 6/15 趋势图内显示的就是 6.15 当日值，不是 6.14 的值。

13. 大屏所有页面的部门筛选项已**剔除**：佳优、佳赏、佳三、佳四 四个货主。

14. **有效 SKU / 有效 SPU / 有效 SKU 日产 数据源**：
   - 实时页面 → 收入表 + 动销表 + 库存表
   - 历史页面 → 收入表 + 动销表

15. **SKU 与货品属性统一从动销表取值**：是否淘汰品、是否瑕疵品、是否停止下单品、商品类型、商品类别、是否单品、是否删除、货品类别、是否自研、是否自产、品牌、品类类目、SKU 与货品归属关系。

16. 「页面昨日指标弹窗」「页面今日指标弹窗」中，最近 30 天趋势图都展示**真实每日值**。

17. **库存 index 表统一限制销售仓 + 代发仓**。
