# 部门排行榜（发货口径）

## 通用规则

- 主表使用货主汇总，限制货主不为空，**默认按本月收入降序**。
- ToC 业务按已付金额核算支付/发货收入；ToB 业务按最新供货价核算。
- **毛利率只统计销售货品**，其他指标统计全部货品的数据。
- 使用 V1 表 left join 库存表获取 V1 表的货主数据；时间范围：dt 从财年初前 30 天到昨日。

## 销售收入类指标

- **今日金额**：`doris_app_real_time_sales_report_rt`，dt
  - 不区分：`sum(sales_amount - refund_amount)`
  - 区分：线上分销(代销)用 `sum(estimate_cost - estimate_refund_cost)`；其他模式用 `sum(sales_amount - refund_amount)`
- **昨日金额（不减/减退款）、昨日自研金额**：`doris_app_real_time_sales_report_v1`，dt；区分逻辑同上。
- **昨日金额占比**：该部门昨日支付（已减退款）/ 所有部门昨日支付（已减退款）。
- **昨日自研金额占比**：该部门昨日自研支付（已减退款）/ 该部门昨日支付（已减退款）。
- **昨日收入(不减退款) / 近30日收入(不减退款)**：`doris_app_report_delivery_v1`
  - 不区分：`sum(sales_amount + refund_amount)`
  - 区分：线上分销(代销) `sum(estimate_brand_quotation + refund_brand_quotation)`；其他 `sum(sales_amount + refund_amount)`
- **7日环比**：(过去7天支付（减退款）− 过去7~14天支付（减退款）)/过去7~14天支付（减退款）×100%；过去7~14 天 = 过去14天 − 过去7天；区分逻辑同 V1 表收入。
- **昨日/近3/7/30/本月/上月/财年/365日 收入 / 自研收入**：`doris_app_report_delivery_v1`
  - 不区分：`sum(sales_amount)`
  - 区分：线上分销(代销) `sum(estimate_brand_quotation)`；其他 `sum(sales_amount)`
  - 实时：列表收入会回刷近 60 天历史；净利含税收入按天回刷，结账后不再回刷。

## 销量与目标完成率

- **今日销量（支付口径）/ 昨日销量（支付口径）**：`doris_app_real_time_sales_report_rt / report`，支付时间，`sum(paid_num - refund_num)`。
- **昨日销量（减退款）**：`doris_app_report_delivery_v1`，dt，`sum(paid_num)`，限制销售货品。
- **月完成率**：部门月度销售收入求和 / 部门月度绩效目标求和。库表：`doris_app_report_delivery_v1` ＋ `das_core.performance_objectives`。
- **年完成率**：部门财年销售收入求和 / 部门财年绩效目标求和。

## 毛利率与采购成本

- **昨日/近30日 含税/不含税毛利率**：1 − (V1 表 含税/不含税采购成本) / V1 表财务销售收入；**限制销售货品**。
- **自研占比 / 收入占比（昨日/近3/7/30/本月/上月/财年/365日）**：V1 表自研销售收入 / V1 表财务销售收入；分子分母均区分销售模式。
- **昨日不含税/含税采购成本**：`doris_app_report_delivery_v1` 直接取 V1 字段，限制销售货品。

## 退款相关

| 指标 | 计算 |
|---|---|
| 昨日未发货退款金额 | `sum(undelivery_refund_amount)` |
| 昨日已发货退款金额 | `sum(refund_amount)` |
| 近30日已发货退款金额 | `sum(refund_amount)` |
| 近30日已发货退款率 | sum(近30日已发货退款) / sum(近30日收入(不减退款)) |
| 近30日未发货退款率 | 近30日未发货退款 / 近30日支付(不减退款) |
| 已发货退款率 | 昨日已发货退款 / 昨日收入(不减退款)；历史用周期口径 |
| 未发货退款率 | 昨日未发货退款 / 支付口径昨日金额(不减退款)；历史用周期口径 |

## 推广费

- **昨日推广费用** = `expense_detail` 支出 − `expense_detail` 收入。
- **本月推广费比** = 本月推广费 / 本月收入；最终汇总至部门。
- **财年推广费比** = 财年推广费 / 财年收入；财年指最近的 4 月 1 日至昨日（不含当天）。
- 库表：`dp_ods.doris_ods_upload_expense_detail` ＋ `doris_app_report_delivery_v1`
- **限定**：`expense_detail` 限制费用项目 cost_code 属于"一级费用类目=推广费用"；限制费用类型=1 实际发生。

## 组织与人力

- **店铺数**：`oms_business.shop`，货主下启用的店铺去重计数（`enabled` 限制为启用）。
- **在岗人数**：钉钉接口落库表
  - 当货主名称=品牌运营部时：二级部门=品牌运营部 且 三级部门=种草组，按 `user_id` 去重计数。
  - 其他部门：二级部门对应人数，按 `user_id` 去重计数。
- **人均日产** = 近30日销售收入 / 30 / 实时在岗人数，**向上取整**；合并的货主需先合并销售收入再除以合并后的在岗人数。
- **人力成本占比** = 1 / 一元人力产出，按百分比保留 2 位（如 56.25%）；客服特殊：`1 / [(集团销售额−电商四部收入) / 成本] × 100%`，集团销售额排除产销事业群的货主。新增"算人力成本占比的上月收入"：1 / 一元人力成本占比 = 1 / (上月收入排除特定科目 / 成本)。计算一元人力产出时排除科目 6051、6301 及其下级，EP001 的收入排除科目 6051、6301（涉及 OW0098 包材一部、OW0068/OW0067/OW0069/OW0076 生产一二三四部、OW0093 模具一部）。1/0 时页面显示 `--`。
- **1元人力产出（元）**：`tmp_data.month_department_employee_cost`（每月初上传一次），date 字段。上月收入 / 人力成本总额，四舍五入保留 2 位；客服上月收入=集团销售额−电四收入，集团销售额排除产销事业群货主收入；各部门人力成本取临时表中最大上传日期对应的部门成本求和。生产一/二/三/四部、模具一部、包材一部用导入的收入/人力成本总额。

## 库存与清仓

| 指标 | 关键逻辑 |
|---|---|
| 库存金额 | 销售仓库存=总库存×不含税月度成本价（为空/0 取不含税含运采购价）；代发仓库存=总库存×不含税含运采购价；仓库功能=1 销售仓 / =2 代发仓，其他不统计 |
| 清仓库存金额 | 同上口径；动销维表 join 库存表，限制 goods_type=1、judge_combine=2 单品、删除状态=0、`item_type`=清仓品；货品类别=销售货品 |
| 实时清仓金额 | `doris_app_real_time_sales_report_rt`，`sum(sales_amount-refund_amount)`，限制商品类别=清仓品（rt 表）、货品类别=销售货品 |
| 昨日/近3/7/30/本月/上月/财年/365日 清仓处理金额 | 销售仓清仓品出库×不含税月度成本价 ＋ 代发仓清仓品出库×不含税含运采购价 ＋ 委外仓清仓品出库×不含税月度成本价；只统计"销售出库 ＋ 分销出库"两种业务模式；库存表字段 `yst_delivery_num`，每天对每天关联 |

## SKU 周转与日产

- **周转天数** = 货主对应的总库存数量 / 加权日均出库数量，**向上取整**。加权日均出库数量 = 7日商品销售出库/7×60% ＋ 14日商品销售出库/14×40%；限制仓库功能=销售仓、代发仓；货品类别=销售货品和配件商品。
- **有效SKU数**：货品类型=销售货品、商品类别=正常品、库存>0；动销 join 库存表（实时取 dt-1 对动销表 dt；历史取区间结束日期 dt 对动销表 dt）；条件：goods_type=1、judge_combine=2、删除=0、货品类别=销售货品、`item_type`=正常品、SKU 正品总库存>0。
- **有效SPU数**：只要有 1 个 SKU 未被打标即为有效货品；条件同上加 `is_valid_goods_with_sale_goods=2`。
- **有效SKU日产** = [近30日支付金额(减退款) / 有效SKU数(昨日)] / 30，**向上取整**；分子不需区分销售模式。
