# 老板报表与 T+1 净利字段逻辑整理

更新时间：2026-07-03

## 一、老板报表

### 1. 全局口径

| 项目 | 逻辑 |
|---|---|
| 近 30 日 | 包含昨天的前 30 天 |
| 本月 | 当月 1 号至昨天，不含当天 |
| 上月 | 上月整月 |
| 财年 | 最近 4 月 1 号至昨天 |
| 业务日期 | 订单/收入通常按发货日期；费用按账单日期或凭证日期 |
| 销售模式 | ToC：线上直销、线下零售直销；ToB：线下分销、平台入仓分销、线上分销、线上代销、贴牌代工；其他归入其他 |
| 国别 | 国内：`country_type=1` 或国家=中国；国外：非中国 |
| 销售模式区分收入 | 线上分销(代销)用供货金额/供货成本；其他模式用销售收入/支付金额 |
| 库存仓限制 | 库存相关统一限制销售仓 + 代发仓，部分清仓处理金额额外含委外仓 |
| 商品属性 | SKU、货品、品类相关属性统一从动销表取 |

### 2. 公司大屏

| 字段 | 数据源 | 计算逻辑 |
|---|---|---|
| 自营仓库存金额 | `doris_dws_SKU_stock_index` | 总库存 × 不含税月末成本价；为空或 0 时取不含税含运采购价；限制奇门仓库、销售仓，排除样品/耗材等货主 |
| 代发仓库存金额 | `doris_dws_SKU_stock_index` | 总库存 × 不含税采购成本单价；限制代发仓 |
| 全仓库存金额 | 库存表 | 自营仓库存金额 + 代发仓库存金额 |
| 清仓品库存金额 | 库存表 + 动销表 | 清仓品库存 × 对应成本价；清仓品包含淘汰品、瑕疵品、停止下单品 |
| 本月清仓处理金额 | 库存表 + 动销表 | 每日 `yst_delivery_num × 成本价` 汇总，按国内/国外区分 |
| 有效 SKU 数 | 动销表 + 库存表 | 商品编码去重；销售货品、单品、未删除、正常品 |
| 有效 SPU 数 | 动销表 | SPU 去重；`is_valid_goods_with_sale_goods=2` |
| 有效 SKU 日产 | 支付表 + 动销表 | 有效 SKU 近 30 日支付金额(减退款) / 有效 SKU 数 / 30，向上取整 |
| 昨日/本月/财年发单量 | 系统订单、跨境订单、CBS 自研订单 | 发货时间范围内系统单号去重 |
| 今日/昨日销售金额 | 实时销售表 | 线上分销(代销)：`estimate_cost - estimate_refund_cost`；其他：`sales_amount - refund_amount` |
| 本月/上月/财年收入 | `doris_app_report_delivery_v1` | 线上分销(代销)：`estimate_brand_quotation`；其他：`sales_amount` |
| 去年/同期收入 | 去年历史表 | 去年对应周期发货收入 - 退款；集团口径需扣产销相关收入 |
| 本年预测收入 | `tmp_date.year_income_rate` | 本年财年收入 + 去年财年收入 × (1 + 同期增长率) - 本年财年收入 |
| 月同比 | V1 + 去年表 | `(上月收入 - 同比月收入) / 同比月收入` |
| 同期增长率 | V1 + 去年表 | `(本财年收入 - 去年财年同期收入) / 去年财年同期收入` |
| 自研销售占比 | V1 | 自研品销售收入 / 总收入；产销事业群需加导入产值 |
| 毛利率 | V1 | `1 - 采购成本 / 财务销售收入`；限制销售货品，分含税/不含税 |

### 3. 部门排行榜

| 字段 | 数据源 | 计算逻辑 |
|---|---|---|
| 今日金额 | `doris_app_real_time_sales_report_rt` | `sales_amount - refund_amount`；线上分销(代销)用 `estimate_cost - estimate_refund_cost` |
| 昨日金额 | `doris_app_real_time_sales_report_v1` | 同今日金额 |
| 昨日金额占比 | 实时 V1 | 部门昨日金额 / 全部门昨日金额 |
| 昨日自研金额占比 | 实时 V1 | 部门昨日自研金额 / 部门昨日金额 |
| 昨日/近 3/7/30/本月/上月/财年/365 日收入 | `doris_app_report_delivery_v1` | 不区分：`sum(sales_amount)`；线上分销(代销)：`sum(estimate_brand_quotation)` |
| 昨日收入不减退款 | V1 | 不区分：`sales_amount + refund_amount`；线上分销(代销)：`estimate_brand_quotation + refund_brand_quotation` |
| 7 日环比 | 实时 V1 | `(过去7天支付 - 过去7~14天支付) / 过去7~14天支付` |
| 今日/昨日销量 | 实时表/V1 | 支付口径：`paid_num - refund_num` |
| 发货销量 | 发货 V1 | `sum(paid_num)`，限制销售货品 |
| 月完成率 | V1 + `das_core.performance_objectives` | 部门月收入 / 部门月度绩效目标 |
| 年完成率 | V1 + 目标表 | 部门财年收入 / 财年绩效目标 |
| 毛利率 | V1 | `1 - 采购成本 / 财务销售收入`，限制销售货品 |
| 采购成本 | V1 | 直接取含税/不含税采购成本字段，限制销售货品 |
| 已发货退款率 | V1 | 已发货退款 / 收入不减退款 |
| 未发货退款率 | 支付表 + V1 | 未发货退款 / 支付金额不减退款 |
| 推广费 | `doris_ods_upload_expense_detail` | 推广费用 = 支出 - 收入，限制一级费用类目=推广费用、实际发生 |
| 推广费比 | 推广费 + V1 | 推广费 / 发货收入(已减退款) |
| 店铺数 | `oms_business.shop` | 启用店铺去重 |
| 在岗人数 | 钉钉人员表 | 二级部门对应人数去重；品牌运营部走种草组特殊口径 |
| 人均日产 | V1 + 人数 | 近30日销售收入 / 30 / 实时在岗人数，向上取整 |
| 人力成本 | `tmp_data.month_department_employee_cost` | 每个货主取最大 `date` 的最新导入成本 |
| 1 元人力产出 | 人力成本表 + 收入 | 普通部门：上月收入 / 人力成本；生产/包材/模具类产销部门：产销导入收入 / 人力成本 |
| 人力成本占比 | 一元人力产出 | `1 / 一元人力产出 × 100%`；分母为 0 时应显示 `--` |
| 周转天数 | 库存表 | 货主总库存 / 加权日均出库，向上取整 |
| 有效 SKU 日产 | 支付表 + 动销表 | 近30日支付金额(减退款) / 有效 SKU 数 / 30，向上取整 |

### 4. 店铺排行榜 / 天猫京东 / 抖音快手 / 拼多多

| 字段 | 数据源 | 计算逻辑 |
|---|---|---|
| 页面范围 | 店铺维度 | 店铺排行榜全平台；天猫京东限定 TB/JD/TBFX；抖音快手限定 DY/KS；拼多多限定 PDD/PDDFX |
| 今日/昨日金额 | 实时表 | `sales_amount - refund_amount`；线上分销(代销)用供货成本减退款成本 |
| 昨日/近30日金额不减退款 | 实时 V1 | `sum(sales_amount)`；线上分销(代销)用 `estimate_cost` |
| 支付销量 | 实时表 | `paid_num - refund_num` |
| 发货销量 | 发货 V1 | `sum(paid_num)` |
| 收入 | 发货 V1 | `sum(sales_amount)`；线上分销(代销)用 `estimate_brand_quotation` |
| 收入占比 | 发货 V1 | 单店铺收入 / 全店铺收入 |
| 自研占比 | 发货 V1 | 自研收入 / 店铺收入 |
| 采购成本 | 发货 V1 | 含税/不含税采购成本求和，限制销售货品 |
| 毛利率 | 发货 V1 | `1 - 采购成本 / 财务销售收入` |
| 推广费/推广费比 | 费用明细 + V1 | 推广费 = 支出 - 收入；推广费比 = 推广费 / 发货收入 |
| 单均价 | 系统订单 + 跨境订单 | 近30日已付金额不减退款 / 原始单量去重 |

### 5. 渠道排行榜

| 字段 | 数据源 | 计算逻辑 |
|---|---|---|
| 渠道归类 | 店铺平台 | 按 `shop_plat_code + shop_type + shop_code` CASE 映射，如 TB=淘宝、DY=抖音、跨境平台归跨境电商 |
| 今日/昨日金额 | 实时表 | `sales_amount - refund_amount`；线上分销(代销)用供货成本口径 |
| 昨日金额占比 | 实时表 | 渠道昨日金额 / 全渠道昨日金额 |
| 近3/7/30/本月/上月/财年/365日收入 | 发货 V1 | `sum(sales_amount)`；排除班牛 |
| 收入占比 | 发货 V1 | 渠道收入 / 全渠道收入 |
| 7日环比 | 实时 V1 | `(过去7天 - 过去7~14天) / 过去7~14天` |
| 毛利率 | 发货 V1 | `1 - 采购成本 / 财务销售收入`；限制销售货品、排除班牛 |
| 自研占比 | 发货 V1 | 自研收入 / 财务销售收入 |

### 6. SKU / 货品 / 自研 / 品类 / 外采排行榜

| 字段 | 数据源 | 计算逻辑 |
|---|---|---|
| 今日销量 | 实时表 | `paid_num - refund_num` |
| 昨日支付销量 | 实时 V1 | `paid_num - refund_num` |
| 发货销量 | 发货 V1 | `sum(paid_num)` |
| 今日/昨日金额 | 实时表 | `sales_amount - refund_amount`；线上分销(代销)用供货成本口径 |
| 收入 | 发货 V1 | `sales_amount`；线上分销(代销)用 `estimate_brand_quotation` |
| 7日环比 | 实时 V1 | `(过去7天支付 - 过去7~14天支付) / 过去7~14天支付` |
| 采购成本 | 发货 V1 | 含税/不含税采购成本求和 |
| 毛利率 | 发货 V1 | `1 - 采购成本 / 收入` |
| 收入占比 | 发货 V1 | 当前对象收入 / 全部对象收入 |
| 自研占比 | 发货 V1 | 自研发货收入 / 发货收入 |
| 件均价 | 发货 V1 | 近30日收入 / 近30日销量，四舍五入取整 |
| 已发货退款率 | V1 | 已发货退款 / 收入不减退款 |
| 未发货退款率 | 支付 V1 + 发货 V1 | 未发货退款 / 支付金额不减退款 |
| 库存金额 | 库存表 | 销售仓库存 × 不含税月度成本价；代发仓库存 × 不含税含运采购价 |
| 可发天数 | 库存表 | 总库存 / 加权日均出库，向上取整 |
| 平均日销 | 库存表 | 7 日出库/7×60% + 14 日出库/14×40% |
| 有效 SKU 数 | 动销表 + 库存表 | 销售货品、单品、未删除、正常品，SKU 去重 |
| 有效 SPU 数 | 动销表 | SPU 去重，`is_valid_goods_with_sale_goods=2` |
| 有效 SKU 日产 | 支付表 + 动销表 | 近30日支付金额(减退款) / 有效 SKU 数 / 30 |
| 可售库存 | 库存表 | 正品总库存 - 正品锁定库存 |
| 在途库存 | 库存表 | `in_transit_stock` |
| 未发库存 | 库存表 | `Undelivered_stock` |
| 可用库存 | 库存表 | 正品总库存 + 在途库存 |
| 可用库存天数 | 库存表 + 支付表 | 可用库存 / 加权日均支付数量 |
| 目标日销/月销/达成率 | `doris_ods_SKU_estimate_sales` | 目标日销求和；目标月销=目标日销×当月天数；达成率=本月销量/目标月销 |
| 含税含运成本单价 | 阶梯价格表/采购价维表 | 优先月度阶梯价最低含税含运价；为空取生效中最低采购价 |
| 快递费 | `doris_dim_SKU` | `max(settlement_freight)` |

### 8. 供应商大屏

| 字段 | 数据源 | 计算逻辑 |

| 产值 | 采购价维表 + 出入库表 | 出库数量/入库数量 × 最新不含税含运采购价，集团供应商展示 |
| 有效 SKU/货品数 | 动销表 | 有效 SKU/SPU 去重 |
| 有效 SKU 日产 | 出库回库表 + 动销表 | 近30日净收入 / 有效 SKU 数 / 30 |
| 件均价 | 出库回库表 | 近30日净收入 / 出库数量，四舍五入取整 |

## 二、T+1 净利

### 1. 净利核心表与数据链路

| 环节 | 表 | 作用 |
|---|---|---|
| 凭证科目中间表 | `doris_dws_voucher_subject_mid` | 凭证 + 凭证明细 + 核算维度 + 科目 + 采集策略生成 |
| 科目费用分摊成功 | `dp_dws.doris_dws_finance_cost_sbjct` | 分摊成功的收入/成本/费用，承接凭证、费用明细、单据费用、薪酬等 |
| 科目费用分摊失败 | `dp_dws.doris_dws_finance_cost_sbjct_failure` | 缺少分摊基础或无法关联维度的数据 |
| 净利 V2 | `doris_app_net_profit_check_report_v2` | 按日期、货主、店铺、SKU、供应商、费用编码输出收入/成本/费用 |
| 供应商采购占比 | `doris_dws_srm_purchase_num_percentage_mid` | SKU 收入、成本、体积分到供应商的主依据 |
| 国内费用明细 | `doris_ods_upload_expense_detail` | 国内费用账单，收入 `income_cost`，支出 `expend_cost` |
| 跨境费用明细 | `expense_detail_cb` | 跨境费用账单，订单关联窗口 3 个月 |

### 2. 费用分摊通用逻辑

| 字段/规则 | 逻辑 |
|---|---|
| `cost_belong` | 1=收入，金额为正；2=成本，金额为负；3=费用，金额为负 |
| `cost_share_method` | 0=按销售收入；1=按销售成本；2=按发货方量 |
| 金额来源 | 国内费用：`income_cost - expend_cost`；跨境同口径 |
| 费用日期 | 费用 detail 用费用发生日期；凭证用业务发生日期；收入成本用发货日期 |
| 订单级费用 | 通过原始订单号关联订单，再按 SKU 收入占比分摊 |
| 店铺级费用 | 按费用发生日期 + 店铺，按该店铺当日 SKU 收入/成本/体积基础分摊 |
| 链接级费用 | 按平台链接/SPU/平台货品关联订单，再按 SKU 收入占比分摊 |
| 订单+链接费用 | 原始订单号 + 平台链接/SPU 共同定位，再按 SKU 占比分摊 |
| 供应商分摊 | 先按 SKU 采购占比拆供应商；没有占比时用来源供应商，仍无则取第一个非瑕疵品供应商 |
| 分摊失败 | 无订单、无 SKU、无收入/成本/体积基础或维表缺失时进入失败表 |

### 3. 净利 V2 字段/EP 逻辑

| EP 编码 | 名称 | 计算/来源 |
|---|---|---|
| EP001 | 含税销售收入/销售收入 | 发货收入已减退款，部分口径来自科目表 |
| EP002 | 含税净利成本 | 含税成本，按下单数量/供应商占比分摊 |
| EP003 | 不含税成本 | `no_tax_product_cost`，按供应商占比分摊 |
| EP004 | 快递自发费 | 旧/含税快递自发费，科目 6601.36 |
| EP005 | 快递代发费/仓储服务费 | 科目 6601.36/6601.51 |
| EP006 | 耗材费 | 科目 6601.62 |
| EP007 | 退货挽损 | 收入 × 分销商损失挽回比例 |
| EP008 | 售后补贴 | 收入 × 分销商售后补贴比例 |
| EP009 | 管理费用税金 | 收入/成本/体积基础 × 店铺税金比例 |
| EP010 | 人工费用 | 上传人工费用按发货日期 + 店铺/货主分摊 |
| EP011 | 分摊费用 | 上传费比或费用金额，费用金额优先 |
| EP012 | 研发费用 | 必须上传费用金额 |
| EP014 | 集团管理费用 | 上传金额，按发货日期 + 店铺分摊 |
| EP015-EP027 | 供应链/宿舍/工位/直播间/影棚/客服/平台佣金/品牌宣传等 | 上传费比或金额；金额优先，按收入/成本/体积分摊 |
| EP028 | 不含税收入 | `sales_amount` 或线上分销供货总额的不含税收入，已减退款 |
| EP029 | 销项税额 | EP001 - EP028 或直接取销项税额 |
| EP030 | 进项税额成本 | EP002 - EP003 或直接取成本税额 |
| EP031 | 不含税卖家运费 | 卖家运费不含税部分，科目 6601.36 |
| EP032 | 不含税仓储服务费 | 科目 6601.51 |
| EP033 | 卖家运费进项税额 | 费用进项税额 |
| EP034 | 代发费进项税额 | 费用进项税额 |
| EP035 | 包装耗材费进项税额 | 费用进项税额 |
| EP036 | 不含税包装耗材费 | 科目 6601.62 |
| EP037 | 其他进项税额费用 | 归入进项税额(费用) |

### 4. 税后净利字段逻辑

| 字段 | 计算逻辑 |
|---|---|
| 不含税收入 | V2 EP028 + 科目费用表中 `cost_belong=1` 的 `no_tax_amount`；产销取导入表 EP001 |
| 不含税成本 | V2 EP003 + 科目费用表中 `cost_belong=2` 的 `no_tax_amount`；产销取导入表 EP003 |
| 不含税费用 | V2 不含税费用 EP + 科目费用表中 `cost_belong=3` 的 `no_tax_amount`；产销取导入表 EP004 |
| 销项税额 | V2 EP029 |
| 进项税额成本 | V2 EP030 |
| 进项税额费用 | 科目费用表 `tax_amount` + 失败表 `tax_amount` + V2 EP033/EP034/EP035/EP037 |
| 应交增值税 | 销项税额 - 进项税额成本 - 进项税额费用 |
| 税金及附加 | 应交增值税 × 12% |
| 水利建设基金 | `(EP028 + 6001.99 + 6051.03) × 0.05%`，国内口径 |
| 印花税买卖合同 | `(EP028 + 6001.99 + 6051.03 + 6401.99 + EP003) × 0.0003` |
| 销售费用无票 | 科目 6601.98 |
| 利润总额 | 不含税收入 - 不含税成本 - 不含税费用 - 税金及附加 - 水利建设基金 - 印花税 |
| 所得税 | 非产销：`max((利润总额 - 税金及附加 - 其他税金 + 销售费用无票) × 15%, 0)`；包含产销时再扣产销不含税收入 |
| 税后净利 | 利润总额 - 所得税 |

### 5. 业务上传临时表规则

| 费用类型 | 上传粒度 | 上传值 | 取数规则 |
|---|---|---|---|
| 分摊费用 | 月 + 店铺 | 费比或费用金额 | 金额优先；只传费比时按基础 × 费比 |
| 人工费用 | 月 + 货主/店铺 | 费用金额 | 当月缺失时取最新上传月份，并除以目标月份天数 |
| 研发费用 | 月 | 费用金额 | 必须上传金额，缺失应失败 |
| 集团管理费用 | 月 + 店铺 | 费用金额 | 月费用 / 目标月份天数，按日期 + 店铺分摊 |
| 供应链管理费/宿舍/工位/直播间/影棚/客服/售后/平台佣金 | 月 + 店铺 | 费比或金额 | 金额优先；费比按收入/成本/体积基础计算 |
| 品牌宣传费 | 月 + 店铺 | 费用金额 | 按上传金额分摊 |

### 6. 国内与跨境差异

| 项目 | 国内 | 跨境 |
|---|---|---|
| 费用明细 | `doris_ods_upload_expense_detail` | `expense_detail_cb` |
| 订单关联窗口 | 通常近 2 个月 | 3 个月 |
| 订单来源 | 系统订单、分销发货单 | 跨境系统订单 index |
| 国别 | 国内 | 国外 |
| 过滤 | 排除跨境平台、班牛、业务公共、产销事业群 | 计算跨境时去掉跨境平台限制，仍排除产销事业群 |
| 订单功能 | 1/2/8 等 | 1 一般交易订单、10 快递使用单等 |

### 7. 排查清单

| 问题类型 | 优先检查 |
|---|---|
| 收入/成本缺失 | V2 是否有 EP028/EP003；发货 V1 是否有来源；店铺/货主维度是否为空或未映射 |
| 费用偏大 | 科目费用分摊表同科目是否多 source/cost_code 重复进入 |
| 分摊偏向某店铺/SKU | 分摊基础是否缺少收入/成本/体积，导致费用集中分摊 |
| 跨境费用异常 | 是否同时取 `expense_detail_cb`、凭证科目、单据费用 index |
| 人工/集团管理费异常 | 上传月份是否缺失；是否取了最新上传月份兜底 |
| 天猫超市异常 | 无收入/成本/体积时是否按发货数量兜底 |
| 税后净利异常 | EP028/EP003、科目表 `no_tax_amount/tax_amount`、EP033-EP037 是否重复或缺失 |
| 产销部门异常 | 是否导入产销收入；产销 1/2/3 部一元人力产出不回退普通发货收入 |
