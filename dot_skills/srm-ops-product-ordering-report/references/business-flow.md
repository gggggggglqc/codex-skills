# 商品版运营订货报表业务全流程

## Purpose

商品版运营订货报表用于回答一个业务问题：某个货主、某个商品、某个仓库是否需要补货，如果需要，是先调拨还是采购，采购多少、向哪个供应商采购、何时到货更合适。

## End-to-End Flow

1. 数据准备
   - 商品范围：销售货品和配件为主，排除不参与商品采购计算的货品。
   - 仓库范围：销售仓、代发仓、工厂仓中符合商品口径的仓；排除 CBS 仓、非销售货品口径仓、部分特殊备注单据。
   - 库存：可用库存、正品库存、残品库存、可销售库存。
   - 在途和未发：已采购未到、采购未发、采购申请中数量。
   - 销售消耗：加权日销、运营预估日销、活动备货消耗。
   - 策略：商品订货策略、货品订货策略、供应商价格、装箱数、再订货点、安全库存等。

2. 生成运营订货报表
   - 按商品、货主、仓库维度计算当前库存能支撑多少天。
   - 判断库存状态：正常、即将不足、缺货、不可售等。
   - 结合策略得到建议采购量。
   - 报表只给采购建议，不等于已经生成采购申请。

3. 报表创建采购申请
   - 用户从报表选择建议行生成采购申请。
   - 系统按供应商、采购组织、货品类型等维度拆分采购申请。
   - 可能直接生成待审核或已审核，取决于是否需要审核及策略/来源逻辑。
   - 报表行会标记已创建采购申请，但采购申请明细不保存原报表行 id。

4. 提交采购申请前调拨
   - 提交时先判断是否开启采购前调拨开关。
   - 商品版调拨是同仓跨货主调拨：目标仓和来源仓相同，来源货主与目标货主不同。
   - 可调拨量来自其他货主在同仓下超过再订货点的可销售库存。
   - 若能调拨，先生成调拨建议；用户确认后，采购申请数量会扣减调拨数量。
   - 如果全部需求都被调拨覆盖，采购申请可被取消；如果部分覆盖，剩余数量继续提交审核。

5. 审核和转采购订单
   - 编辑中可以提交；待审核可以审核、驳回、撤销。
   - 已审核后，未取消、未转单、数量大于 0、有有效采购价的明细可以转采购订单。
   - 转单后进入采购订单、交货计划、发货、到货、入库等后续链路。

## Core Calculation Concepts

### Inventory Turnover KPI

库存周转 = 正品总库存货值可周转天数。

可发天数 = 总库存数量 / 加权日均出库数量，结果向上取整。

总库存 = 正品总库存 + 残品总库存。

加权日均出库数量 = 近 7 日商品销售出库数量 / 7 * 60% + 近 14 日商品销售出库数量 / 14 * 40%。

业务限制：
- 仓库限制为销售仓和代发仓。
- 货品类型限制为销售货品和配件。
- 跨境只统计跨境自营仓库存周转，未统计货代中转仓和海外仓。
- 京东自营库存数量不准，销售数量月度生成，无法正常使用库存周转。
- 分销按正常口径使用。

### Daily Sales

加权日销体现近期真实销售消耗，7 日权重更高，14 日权重用于平滑短期波动。

运营预估日销用于人工修正未来需求，适合新品、活动前、渠道变化、短期异常销售等场景。讲业务时要说明：系统日销看历史，运营预估日销看未来判断。

### Reorder Point

再订货点是触发补货的库存警戒线。库存消耗到再订货点附近时，需要启动采购，因为供应商到货存在周期。

经典解释图应表达：
- 库存随销售持续下降。
- 到达再订货点时下采购单。
- 供应商到货前库存继续消耗。
- 到货时库存最好接近 0 或安全库存，不要断货也不要过量。

### Suggested Purchase Quantity

业务表达：
建议采购量 = 目标备货需求 - 当前可用供给。

当前可用供给通常包括：
- 可销售库存。
- 已采购在途。
- 采购未发。
- 采购申请中。
- 可调拨库存。

目标备货需求通常由策略决定：
- 覆盖多少天销售。
- 是否加安全库存。
- 是否考虑活动备货。
- 是否按装箱数取整。
- 是否受最小采购量、供应商交期、采购价维护等限制。

## Strategy Effects

### 商品订货策略

商品订货策略决定某个商品在某个业务场景下是否进入订货计算，以及使用什么补货目标。

讲解时重点覆盖：
- 是否启用：未启用或不匹配的商品不进入计算。
- 适用范围：按货主、仓库、商品/货品、供应商、采购组织等维度匹配。
- 备货天数：决定希望补到多少天的销售量。
- 再订货点：决定何时触发采购建议。
- 安全库存：让库存不只补到 0，而是保留缓冲。
- 活动备货：活动需求会额外占用库存或提高建议采购量。

### 货品订货策略

货品订货策略更偏向商品自身的采购约束和供应侧约束。

讲解时重点覆盖：
- 供应商交期：决定从下单到到货之间需要覆盖多少销售消耗。
- 最小采购量或起订量：建议采购量小于起订量时可能被抬高。
- 装箱数：最终采购数量可能按整箱向上或向下调整。
- 采购价维护：没有有效采购价时不能正常转采购订单。
- 内部供应商绑定仓：内部供应商在部分工厂/代发仓存在绑定校验，不匹配时不能提交或转单。

## Internal vs External Supplier

内部供应商：
- 更像集团内部供给，可能绑定特定工厂仓或代发仓。
- 提交和转单时会校验内部供应商与仓库绑定关系。
- 交期存在最大天数限制，特殊 KJ 商品或虚拟/暂存仓可例外。

外部供应商：
- 按正常采购供应商处理。
- 重点校验供应商启用状态、采购价、阶梯价、交期、起订量等。
- 不走内部供应商绑定仓限制。

## Purchase Application Status and Viewing

状态：
- 编辑中：可修改、取消、提交。
- 待审核：可审核、驳回、撤销。
- 已审核：可转采购订单，可取消未转单明细。
- 已取消：通常不再继续业务操作。

查看：
- 列表和明细列表可按状态筛选。
- 不传状态时，后端不默认限制状态。
- 详情按单号查询，不因状态限制查看。

## PPT / Training Guidance

Use this slide structure for business training:

1. 考核指标：库存周转天数
   - 用公式讲清楚为什么要控制库存周转。
   - 单独写限制范围和特殊业务口径。
2. 业务目标：既不断货，也不积压
   - 用库存波动图展示再订货点、供应商到货周期、到货后库存回升。
3. 哪些数据参与计算
   - 库存、日销、运营预估日销、在途、未发、申请中、活动备货、策略。
4. 策略如何改变结果
   - 商品订货策略决定是否算、何时算、补到多少。
   - 货品订货策略决定供应商、交期、起订、整箱、价格。
5. 采购前为什么先调拨
   - 同仓其他货主有多余库存时，先调拨再采购。
   - 调拨覆盖多少，采购申请就减少多少。
6. 从报表到采购订单
   - 报表建议 -> 创建采购申请 -> 提交前调拨 -> 审核 -> 转采购订单。

Avoid:
- Do not display Java field names to business users.
- Do not stack many modules on one slide.
- Do not use only text; use wave/stock diagrams for reorder point and arrival-cycle effects.
