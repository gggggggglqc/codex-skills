## Usage

Use these skills as small, composable engineering workflows. The most commonly useful ones are:

- `mattpocock-grill-with-docs`: clarify a code change before implementation and update project context docs.
- `mattpocock-diagnose`: debug a hard bug through reproduce, minimise, hypothesise, instrument, fix, and regression-test.
- `mattpocock-tdd`: build or fix with a red-green-refactor loop.
- `mattpocock-to-prd`: turn conversation context into a PRD.
- `mattpocock-to-issues`: split a plan into independently actionable issues.
- `mattpocock-handoff`: create a compact handoff for another agent/session.

Run `mattpocock-setup-matt-pocock-skills` once per code repo when you want the full issue-tracker and domain-document workflow.
