# T+1 定期对数流程

## 用途

本 skill 沉淀了 T+1 定期对数的完整操作流程，来源为《T+1对账流程.xlsx》，覆盖以下六大模块：

1. **结账检核表**：4 日/6 日凭证与业务报表核对项
2. **月初工作流程**：每月 1 日 ~ 15 日逐日任务、负责组别、负责人、完成时间
3. **费用入账规范**：净利表上传类、收入/成本/其他收入支出、推广/平台/售后/信息服务、每刻报销/自动计算/进销存
4. **计提费用**：冲销上月计提 → 实际账单按日重分摊 → 次月计提
5. **日常事务**：出纳单据/每刻单据下推凭证、进销存推送凭证、业务报表凭证检查
6. **问题答疑 + 差异排查清单**

## 何时使用本 skill

当用户提到下列关键字时，**必须**先按需读取参考文档再回答：

- T+1 对数 / 对账 / 结账 / 刷数 / 重跑数据
- 几号做什么 / 月初任务 / 排期 / 时间节点 / 负责人
- 费用入账 / 费用导入 / 入账规范 / 取数表
- 计提 / 冲销 / 分摊 / 预提
- 出纳单据 / 每刻单据 / 下推凭证 / 凭证导入
- 凭证排除 / 导入排除 / 剔除部门 / 剔除店铺
- 差异排查 / 数据对不上 / BI 和弹窗不一致 / 费用和账单对不上
- 换货主 / 跨月退款 / 退货继承

> 如果问题涉及**净利核算口径、EP 编码、分摊公式**，应同时参考「老板报表T1净利」skill。

## 计提费用规则

### 处理原则

1. **先冲销**上月计提费用到月底最后一天。
2. **实际账单出来后**，按实际账单重新按天分摊。
3. 次月费用需在 **8 号前**完成计提分摊。

### 凭证导入排除规则

**凭证导入需剔除的部门**：总裁办、财务中心、人力行政中心、研销中心、信息中心。

**凭证导入需剔除的店铺**：业务公共、佳帮手分销旗舰店XX【国内物流部】、绩效福利兑换XX【人力行政部】、银行业务渠道店XX【财务中心】、银行线下批量采购店XX【财务中心】、跨境店铺、产销店铺。

**出纳单据下推排除**：
- 剔除部门：总裁办、人力行政中心、财务中心、研销中心、信息中心、仓储配送部、产销、电商四部-航天仓
- 剔除店铺：跨境
- 剔除费用项目：不涉及损益科目

**每刻单据下推规则**：
- 只推送国内店铺维度费用
- 排除磁力金牛/营销工具线下/信息服务费等需按摊销规则判断的项目
- 其他业务付款单排除推广其他线上
- 运杂费排除京东自营店铺（佳帮手京东清洁自营店/收纳自营店/厨具自营店/清洁京东自营旗舰店等）

**进销存推送凭证删除规则**：
- 删除店铺：佳帮手分销旗舰店XX【国内物流部】、绩效福利兑换XX【人力行政部】
- 操作路径：业务报表-凭证详情-反审核；财务-进销存管理-销售业务发货-删除业务报表凭证

## 差异排查 10 条清单

1. **确认刷数批次**：5 号凌晨 3 点 / 6 号整月刷数 / 7 号凌晨 3 点重跑或刷上月最后一天。
2. **确认涉及数据表**：V2 净利表、科目分摊成功表、科目分摊失败表、费用账单、财务凭证、费用采集策略表。
3. **检查凭证状态**：业务报表账套凭证是否已审核/已结账，是否存在下推失败。
4. **核对出纳单据**：手工录入/批量导入/单据复制的金额是否与集团核算一致。
5. **检查重复上传**：仓储服务费/运杂费/卖家运费/返利费是否重复，上传人员是否为往来组。
6. **检查采集策略**：手工导入/批量导入/凭证复制的科目是否为采集策略中"是"的科目。
7. **费用账单 vs BI**：优先确认税率拆分、不含税口径、费用归属和正负号。
8. **净利导入类费用**：是否按 6 号 12 点前提供，是否按店铺/月份/费用金额或费比口径上传。
9. **计提类费用**：是否已冲销上月计提，并按实际账单重新按日分摊。
10. **换货主/退款跨月**：区分下单日期、发货日期、退款继承原订单货主。

## 问题答疑 FAQ

| 问题 | 解答 |
|---|---|
| 费用账单税率按什么规则拆分？ | 查看费用项目主数据的税率拆分规则 |
| 费用项目如何判断收入/费用/成本？ | 根据费用采集策略判断分摊方式及金额加减法 |
| 费用账单和 BI 对不上？ | 先检查是否使用不含税费用，是否需要去税 |
| BI 费用和弹窗费用为什么取反？ | BI 曾对费用项取反（不含税收入取正），后续已修改为和弹窗一致 |
| 所有费用账单都是负数减正数差额入账？ | 要看费用编码和拆税规则，售后返款是收入拆税，平台佣金是费用拆税 |
| 其他税金算哪个指标？ | 包含水利建设基金和印花税-买卖合同 |
| 有收入没成本的原因？ | 销售订单退款但非全退，成本已抵消但退款金额存在差额 |
| 换货主后为什么还显示旧部门？ | 正向订单按下单时货主继承；退款订单继承原发货订单货主 |

### 版本更新规则

1. **版本号**：与禅道版本号保持一致（如 V1.0.0、V1.1.0、V2.0.0）
2. **Sheet 命名**：指标说明按"版本号 + 简述"格式（如 `V1.1.0 口径调整`）
3. **变更标识**：本期新增或修改内容使用**红色字体**标识；删除内容使用~~画线~~处理
4. **历史版本**：只隐藏不删除，便于追溯
5. **额外说明**：绑定版本号，格式为 `【V1.x.0 额外说明】`
6. **更新流程**：复制上一版本 → 修改本期内容（标红）→ 更新版本说明 → 检查额外说明 → 隐藏历史版本 → 上线后更新状态
