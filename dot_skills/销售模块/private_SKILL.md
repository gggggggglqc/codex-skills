## 指标体系总览

### 四层单据结构

```
原始订单（平台抓取）→ 系统订单（ERP处理后）→ 分销订单（DMS）→ 分销发货单（DMS）
```

每层单据都有**单据级指标**（订单维度）和**商品级指标**（SKU维度）两种粒度。

### 核心指标关系

```
货款 = 单价 × 数量
应收金额 = 货款 − 商家优惠 − 商家分摊优惠 + 分摊邮费
已付金额 = 买家支付 + 平台补贴 + 支付工具优惠
净利成本（不含税）= 净利成本（含税）− 进项税额（成本）
```

### 成本分层查询逻辑

系统订单商品的采购成本和净利成本，按**发货仓库功能**（销售仓/暂存仓/代发仓/委外仓）采用不同的分层回退查询策略：

1. 优先查存货成本表（月末加权成本价）
2. 回退到品牌供货价-标准价（西安仓优先）
3. 回退到采购价格表（含税/不含税含运费单价）
4. 最终回退到月度采购阶梯价格表

### 销售模式判定规则（优先级从高到低）

| 优先级 | 销售模式 | 核心判定条件 |
|---|---|---|
| 2.1 | 非销售业务 | 补发货品/配件/样品/快递取号/达人样品 |
| 2.2 | 线上直销 | 自营网店 + 主流电商平台 |
| 2.2 | 线上分销（代销） | 外部客户网店/供销店铺 + 代销类型 |
| 2.2 | 线上分销（经销） | 供销店铺 + 经销类型 |
| 2.2 | 平台入仓分销 | DMS分销单 + 非线下平台 |
| 2.2 | 线下零售/直销 | 线下零售 + 线下/微信社群平台 |
| 2.3 | 线下分销 | DMS分销单 + 线下平台 |
| 2.4 | 贴牌代工 | DMS贴牌代工 + 线上平台 |

## 使用约定

1. 回答指标问题前，**先确认指标所属单据层级**（原始订单/系统订单/分销订单/分销发货单），不同层级的同名指标计算逻辑可能不同。
2. 引用修饰词时，**带上面向开发释义中的具体判定条件**（如 OMS 系统订单的字段条件、DMS 分销订单的字段条件），避免给出模糊答案。
3. 成本类指标必须说明**是否含税**以及**对应的仓库功能分层逻辑**。
4. 跨境销售指标涉及币种转换时，统一按**订单支付日期当天的汇率表**将外币转为人民币。
5. 精度要求：数量类指标为整数，金额类指标保留小数点后 5 位。

### 版本更新规则

1. **版本号**：与禅道版本号保持一致（如 V1.0.0、V1.1.0、V2.0.0）
2. **Sheet 命名**：指标说明按"版本号 + 简述"格式（如 `V1.1.0 口径调整`）
3. **变更标识**：本期新增或修改内容使用**红色字体**标识；删除内容使用~~画线~~处理
4. **历史版本**：只隐藏不删除，便于追溯
5. **额外说明**：绑定版本号，格式为 `【V1.x.0 额外说明】`
6. **更新流程**：复制上一版本 → 修改本期内容（标红）→ 更新版本说明 → 检查额外说明 → 隐藏历史版本 → 上线后更新状态
