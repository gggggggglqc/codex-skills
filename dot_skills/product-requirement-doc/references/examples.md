# Examples

## Backend-Only Draft

Title:

```text
【后端】销售业务日志移除工厂数据；合并推送无匹配分组策略时跳过数据
```

Product design:

```text
一.修改功能点

（1）销售业务日志移除工厂数据
1.调整：销售业务日志查询及写入时，过滤属于工厂的数据，避免进入销售业务日志范围。
2.删除：删除工厂数据在销售业务日志中的展示或处理逻辑。

（2）合并推送无匹配分组策略时跳过数据
1.新增：无匹配数据分组策略时的跳过处理。
2.调整：合并推送流程中，未匹配到分组策略的数据不再继续推送。

二.历史数据处理方法
1.历史数据范围：待确认是否包含已生成销售业务日志和已推送数据。
2.处理方式：待确认是否需要清理或仅新逻辑生效。
3.执行时机：待确认。
4.校验方法：通过抽样对比工厂数据过滤结果和合并推送日志验证。
```

## Frontend And Backend Draft

Title:

```text
【前端+后端】新增导入结果明细；调整失败原因展示；删除旧版导入提示
```

Product design:

```text
一.修改功能点

（1）导入结果明细
1.新增：新增导入成功、失败、跳过数量及明细查看能力。
2.调整：后端接口返回导入结果明细字段，前端按结果状态分类展示。

（2）失败原因展示
1.调整：失败原因展示为业务可读文案，并支持按失败类型筛选。
2.删除：删除旧版统一失败提示。

二.历史数据处理方法
1.历史数据范围：仅影响新导入任务，历史导入记录不补充明细。
2.处理方式：无历史数据处理。
3.执行时机：上线后生效。
4.校验方法：通过新增导入任务验证明细字段和页面展示。
```
