# 存货成本核算规则

## 使用流程

1. 先识别核算范围：
   - 用户提到“工厂存货成本计算逻辑3.1”“工厂”“物料仓”或工厂仓场景时，使用工厂规则。
   - 用户提到“集团存货成本计算逻辑3.1”“集团”、销售退货、样品货主、分销或集团货主维度场景时，使用集团规则。
   - 如果用户没有说明范围，并且工厂与集团答案会明显不同，再追问是工厂还是集团。

2. 再识别单据：
   - 优先按单据类型匹配；如果有入库原因或出库原因，再叠加原因匹配，例如 `其他入库单 + 项目组自采`、`其他出库 + 库存调整`。
   - 保留特殊业务口径，尤其是“本月销售出本月退货入”“非本月销售出本月退货入”“货主=样品”“自研品”“非自研品”和调拨月份限制。

3. 当问题需要具体规则取值时，读取 [references/cost-rules.md](references/cost-rules.md)。不要只回答摘要，必须优先查“逐单据规则台账”和“价格取不到兜底规则”。

4. 回答时优先给出与问题相关的规则字段：
   - 是否参与出库成本价计算/参与出库成本计算
   - 是否参与结存成本计算
   - 成本来源名称
   - 取值逻辑或赋值逻辑
   - 参与月末成本价计算的库存数量
   - 参与成本价计算的状态
   - 是否参与收入计算/收入总额计算
   - 是否参与发出总额计算
   - 成本价赋值时机
   - 样品货主或其他特殊说明

5. 如果把规则转换成 SQL、代码或对账逻辑：
   - 将“数量金额不参与”和“无此类单据/物料仓无此单据”视为明确排除规则。
   - 对“免费配件入库”“特殊品入库”等零成本规则，按“数量参与、单价为 0”处理；除非规则另有说明，不要当成排除。
   - 将“负值参与收入/发出/出库计算”视为抵消规则；除非目标系统明确要求符号归一，否则不要转成正数。
   - 对“最新采购价”的平均值，规则要求排除成本价为 0 的记录时，必须先排除再平均。
   - 对调拨单专用的最新采购价规则，规则要求“调出月份之前”的，必须限制取价记录早于调出月份。
   - 如果按优先级取不到价格，继续执行“价格取不到兜底规则”；兜底规则仍取不到时，按规则记录为空或 0，不能自行编造价格。

## 必须覆盖的规则颗粒度

回答或实现规则时，必须按“核算范围 + 单据类型 + 入库/出库原因 + 场景条件”定位到具体规则。

逐单据规则至少要保留以下条件：
- 参与出库成本价计算或参与出库成本计算。
- 是否参与结存成本计算。
- 成本来源名称。
- 取值逻辑或赋值逻辑。
- 参与月末成本价计算的库存数量。
- 参与成本价计算的状态。
- 是否参与收入计算或收入总额计算。
- 是否参与发出总额计算。
- 成本价赋值时机。
- 样品货主、销售当月退货、归还、调拨跨月/同月、清仓品/残次品、物料仓无单据等特殊条件。

## 回答格式

单个单据优先使用紧凑规则卡片：

```text
范围：集团
单据：退货应收入库单
参与：分场景
成本来源：本月销售出本月退货入=无需成本；非本月销售出本月退货入=上游单据携带；剩余=入库钩稽
取值：本月销售出本月退货入不取值；非本月销售出本月退货入取销售出库单成本价；剩余取优先上月出库成本价
```

如果是对比多个范围或多个单据，使用表格。保留原始中文业务术语，不随意改写字段名和规则口径。
