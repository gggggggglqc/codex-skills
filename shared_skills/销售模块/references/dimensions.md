# 维度概念

## 业务实体类型

| 维度 | 释义 |
|---|---|
| 平台 | 电商平台，如淘宝、拼多多、抖音、京东等 |
| 店铺 | 电商平台上的店铺，如佳邦手旗舰店 |
| 仓库 | 储存货物并根据订单信息完成发货的地点 |
| 物流名称 | 物流类别下各城市具体的物流网点 |
| 物流类别 | 物流的类别，如邮政、顺丰、圆通、申通、极兔等 |
| 货主 | 货品的所属方 |
| 供应商 | 提供货品及相应服务的企业及其分支机构、个体工商户 |
| 分销商 | 获得商品所有权的中间商，通过购买取得商品所有权并转售 |
| 达人 | 在抖音或快手上帮忙带货的达人 |

## 组织类型

| 维度 | 释义 |
|---|---|
| 部门 | 公司内的部门 |
| 事业群 | 公司内部业务部门划分，目前分为品牌事业群、电商事业群，只与货主有对应关系 |
| 公司 | 经营工商业企业联合的团体 |
| 集团 | 由众多关联企业组成的联合体，集团下有多个下属公司 |

## 货品类型

| 维度 | 释义 |
|---|---|
| 平台货品ID | 电商平台上销售的链接货品 |
| 平台规格ID | 电商平台上销售的链接货品的规格 |
| 货品/SPU | 商品信息聚合的最小单位，如"厨房餐边柜80款-4层" |
| 单品/SKU | 商品的销售属性集合，供买家下单时点选 |
| 组合装 | 多个单品组合销售 |
| 一级类目 | 商品隶属的一级类目 |
| 二级类目 | 商品隶属的二级类目 |
| 三级类目 | 商品隶属的三级类目 |
| 四级类目 | 商品隶属的四级类目 |
| 品牌 | 商品的品牌 |

## 地域类型

| 维度 | 释义 |
|---|---|
| 省份 | 自然地理位置的省 |
| 城市 | 自然地理位置的市 |
| 区县 | 自然地理位置的区或县 |

## 人员类型

| 维度 | 释义 |
|---|---|
| 创建人 | 单据/实体的创建人 |
| 审核人 | 单据的审核人 |
| 用户ID | 用户标识 |

## 维度关联规则

以下维度不是直接取字段值，需要通过编码关联维表获取：

| 目标维度 | 关联方式 |
|---|---|
| 店铺名称 | 通过店铺编码关联店铺维表 |
| 店铺平台编码/名称 | 通过店铺编码关联店铺维表 |
| 店铺类型 | 通过店铺编码关联店铺维表 |
| 仓库名称 | 通过仓库编码关联仓库维表 |
| 货品编码/名称 | 通过SKU编码关联SKU维表 |
| 货品类别 | 通过SKU编码关联SKU维表获取货品编码 |
| 一级~四级类目名称 | 通过SKU编码→多级类目ID→类目维表 |
| 是否自研品/控价品/淘汰品 | 通过SKU编码关联SKU维表 |
| 国家英文名称 | 通过国家二字码关联国家维表 |
| 订单平台名称 | 通过平台编码关联平台维表 |
